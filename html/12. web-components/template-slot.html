<template> & <slot>

<template> — Inert Markup

Content inside <template> is parsed but not rendered. It's a reusable blueprint you clone and insert via JavaScript.

Cards stamped from a <template>

<template id="card-template">
  <div class="card">
    <img class="card-img">
    <h3 class="card-title"></h3>
    <p  class="card-desc"></p>
  </div>
</template>

<script>
  const tpl = document.getElementById('card-template');

  function addCard({ title, desc, img }) {
    const clone = tpl.content.cloneNode(true); // deep clone
    clone.querySelector('.card-title').textContent = title;
    clone.querySelector('.card-desc').textContent  = desc;
    clone.querySelector('.card-img').src            = img;
    clone.querySelector('.card-img').alt            = title;
    container.appendChild(clone);
  }
</script>

<slot> — Placeholders in Shadow DOM

<slot> is used inside a Web Component's Shadow DOM template. It marks where the caller's light DOM children are projected.

Pro Tip Named slots let callers inject content into specific positions of a component. Warning Shadow DOM styles are scoped — they don't leak in or out of the component boundary. Default Box When no type is provided, the component uses its default style. 
<!-- Usage -->
<info-box type="tip">
  <span slot="title">Pro Tip</span>
  <span slot="body">Named slots let callers inject content...</span>
</info-box>

<!-- Component definition (Shadow DOM template with slots) -->
<template id="info-box-tpl">
  <style>...</style>
  <div class="box">
    <strong><slot name="title">Info</slot></strong>
    <p><slot name="body"></slot></p>
  </div>
</template>
