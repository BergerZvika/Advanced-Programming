<picture> & Responsive Images

<picture> — Art Direction

<picture> lets you serve different image sources based on viewport size or format support. The browser picks the first matching <source>; the <img> is the required fallback.

Resize the window below 600 px to see the portrait crop load instead. 

<picture>
  <!-- Portrait crop on mobile -->
  <source media="(max-width: 600px)"
          srcset="photo-portrait.avif" type="image/avif">
  <source media="(max-width: 600px)"
          srcset="photo-portrait.webp" type="image/webp">
  <source media="(max-width: 600px)"
          srcset="photo-portrait.jpg">

  <!-- Landscape on desktop -->
  <source srcset="photo-landscape.avif" type="image/avif">
  <source srcset="photo-landscape.webp" type="image/webp">

  <!-- Fallback -->
  <img src="photo-landscape.jpg" alt="Scenery" width="800" height="300">
</picture>

srcset + sizes — Resolution Switching

When the content is the same but you want the browser to pick the right file size, use srcset + sizes on a plain <img>.

<img
  src="photo-800.jpg"
  srcset="photo-400.jpg   400w,
          photo-800.jpg   800w,
          photo-1600.jpg 1600w"
  sizes="(max-width: 600px) 100vw,
         (max-width: 900px) 80vw,
         760px"
  alt="Photo"
  width="800" height="400"
  loading="lazy">
ConceptHow it works
srcset with w descriptors List image files and their actual pixel widths. Browser knows available options.
sizes Tell the browser how wide the image will render at various breakpoints. Without this it assumes 100vw.
Browser choice Browser picks the best candidate based on rendered size × device pixel ratio (DPR). A 2x Retina screen at 400px may load the 800w image.

2x / 3x Pixel Density (x descriptors)

For fixed-size images (icons, logos), use x descriptors to serve higher-res versions to high-DPI screens:

<img
  src="logo.png"
  srcset="logo.png 1x, logo@2x.png 2x, logo@3x.png 3x"
  alt="Company Logo"
  width="120" height="40">

Lazy Loading Strategy

<!-- Images below the fold -->
<img src="below-fold.jpg" alt="..." loading="lazy">

<!-- Hero / LCP image — load eagerly (default) + preload in <head> -->
<link rel="preload" as="image" href="hero.jpg">
<img src="hero.jpg" alt="Hero" loading="eager" fetchpriority="high">

Format Support Detection with <picture>

Use <picture> to progressively serve modern formats, falling back to JPEG:

<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Photo" width="800" height="600">
</picture>
