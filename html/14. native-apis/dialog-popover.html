<dialog> & Popover API

<dialog> — Native Modal

The native <dialog> element provides focus trapping, Escape to close, and a ::backdrop pseudo-element — no library needed.

Information

This is a native <dialog> element. It handles:

  • Focus trap (Tab stays inside)
  • Escape key to close
  • ::backdrop overlay
  • Correct inert state for background content

Quick Edit

 
<dialog id="my-dialog">
  <h2>Title</h2>
  <p>Content...</p>
  <button onclick="document.getElementById('my-dialog').close()">Close</button>
</dialog>

<!-- Open as modal (with backdrop + focus trap) -->
<button onclick="document.getElementById('my-dialog').showModal()">Open</button>

<!-- Or as non-modal (no backdrop) -->
<button onclick="document.getElementById('my-dialog').show()">Open inline</button>

<!-- Listen for close + return value -->
<script>
  dialog.addEventListener('close', () => {
    console.log(dialog.returnValue); // value passed to .close()
  });
</script>

Popover API

Popovers are non-modal overlays that sit in the top layer (above everything, no z-index needed). They close on click-outside and Escape automatically.

Popover API

This uses popover attribute. Click outside or press Escape to dismiss. No JS required for basic open/close.

popover="auto" closes on outside-click. popover="manual" requires explicit JS to close.

<!-- Declarative: button targets a popover by ID -->
<button popovertarget="my-pop">Open</button>
<div id="my-pop" popover>Hello!</div>

<!-- Manual popover: JS-controlled only -->
<div id="notif" popover="manual">Saved!</div>
<script>
  const pop = document.getElementById('notif');
  pop.showPopover();
  setTimeout(() => pop.hidePopover(), 3000);
</script>

dialog vs. Popover — When to use which

<dialog> Popover API
Modal? Yes (with showModal()) No
Focus trap Yes No
Backdrop Yes (::backdrop) No (transparent)
Light dismiss Esc only Esc + click outside
Best for Confirmations, forms, alerts requiring user action Tooltips, dropdowns, notifications, non-blocking overlays
