datalist, output, meter, progress

<datalist> — Autocomplete suggestions

<datalist> provides a suggestion list for an <input>. The user can still type anything — it's not a dropdown restriction.


<input type="text" list="cities-list" id="city">
<datalist id="cities-list">
  <option value="Berlin">
  <option value="London">
  <option value="Paris">
</datalist>

<output> — Computed Result

<output> is a semantic element for displaying the result of a calculation. Its for attribute points to input IDs it depends on.

BMI: —

<form oninput="result.value = calcBMI(weight.value, height.value)">
  <input type="number" id="weight" name="weight">
  <input type="number" id="height" name="height">
  <output name="result" for="weight height"></output>
</form>

<meter> — Scalar Gauge

<meter> represents a known range — disk usage, score, rating. Browsers apply semantic color (green / yellow / red) based on low, high, and optimum.

<meter value="75" min="0" max="100"
       low="50" high="80" optimum="20"></meter>
AttributeMeaning
valueCurrent value
min / maxRange boundaries
lowThreshold below which values are "low"
highThreshold above which values are "high"
optimumThe preferred value — determines which zone is "good"

<progress> — Task Progress

<progress> shows completion of a task. Omit max and value for an indeterminate (spinning) state.

<!-- Determinate -->
<progress value="65" max="100"></progress>

<!-- Indeterminate -->
<progress></progress>
