Semantic HTML Elements

Semantic elements tell the browser (and screen readers, search engines, and developers) what a section is — not just how it looks.

Page Layout Elements

<header>
<nav>
<main>
  <article>
    <section>
<aside>
<footer>

Element Reference

ElementUse it for
<header>Site or section header — logo, site title, top nav. Can appear inside <article> too.
<nav>Major navigation links. Don't wrap every link group — only primary nav menus.
<main>The primary content of the page. Only one per page.
<article>Self-contained content that makes sense on its own: blog post, news story, forum thread, card.
<section>Thematic grouping of content. Should have a heading (<h2>+).
<aside>Content tangentially related to the main content: sidebar, pull quote, ad, related links.
<footer>Page or section footer — copyright, links, contact info. Can appear inside <article>.
<address>Contact information for the nearest <article> or the page author.
<time>Machine-readable date/time: <time datetime="2024-01-15">January 15</time>
<figure> + <figcaption>Image, diagram, or code with an optional caption.
<details> + <summary>Native disclosure widget — expand/collapse without JavaScript.
<dialog>Native modal/dialog (use JS .showModal() to open).
