The <head> Element & Meta Tags

Everything in <head> is metadata — not rendered as visible content.

Essential Tags

TagPurpose
<meta charset="UTF-8">Character encoding — always the first tag in <head>
<meta name="viewport" ...>Makes the page responsive on mobile (required for all sites)
<title>Tab title and search engine headline — keep it under 60 chars
<meta name="description">Snippet shown below the title in search results — 150-160 chars

Viewport Meta

<meta name="viewport" content="width=device-width, initial-scale=1.0">
  • width=device-width — match the device's screen width
  • initial-scale=1.0 — don't zoom in/out on load
  • Without this, mobile browsers render pages at ~980px and shrink them — text becomes tiny

Open Graph Tags (Social Previews)

When someone shares your URL on Facebook, LinkedIn, Slack, etc., these tags control the preview card:

<meta property="og:title" content="Page Title">
<meta property="og:description" content="Short description.">
<meta property="og:image" content="https://example.com/thumb.jpg">
<meta property="og:url" content="https://example.com/page">
<meta property="og:type" content="website">

Script Loading

<!-- Blocks HTML parsing — avoid for most scripts -->
<script src="app.js"></script>

<!-- Downloads in parallel, executes when ready (order not preserved) -->
<script src="app.js" async></script>

<!-- Downloads in parallel, executes after HTML is parsed (order preserved) -->
<script src="app.js" defer></script>

Prefer defer for most scripts. Use async only for independent scripts (analytics, ads).
