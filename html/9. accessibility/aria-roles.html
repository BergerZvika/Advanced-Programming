ARIA Roles & Attributes

ARIA (Accessible Rich Internet Applications) lets you expose the meaning of custom widgets to assistive technologies when native HTML semantics aren't enough.

Rule #1: Use native HTML elements first. ARIA is for when no native element fits.

Core ARIA Attributes

AttributePurposeExample
roleOverride or add a semantic rolerole="button", role="alert"
aria-labelProvide an accessible name (no visible text needed)aria-label="Close dialog"
aria-labelledbyPoint to another element as the labelaria-labelledby="heading-id"
aria-describedbyPoint to an element with a longer descriptionaria-describedby="hint-id"
aria-hiddenHide element from assistive tech (decorative icons)aria-hidden="true"
aria-expandedState of a collapsible regionaria-expanded="false"
aria-selectedSelected state (tabs, listboxes)aria-selected="true"
aria-checkedChecked state of custom checkboxes/radiosaria-checked="true"
aria-disabledDisabled state without removing from tab orderaria-disabled="true"
aria-liveAnnounce dynamic content updates to screen readersaria-live="polite"
aria-atomicRead the entire region on update, not just changed partaria-atomic="true"
aria-requiredMark field as required (use HTML required instead when possible)aria-required="true"
aria-invalidSignal a validation erroraria-invalid="true"
aria-controlsPoint to the element this control affectsaria-controls="panel-id"
aria-currentCurrent item in a navigationaria-current="page"

Screen-reader-only Text

The .sr-only pattern hides text visually but keeps it accessible:

.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
}

Use case: an icon button with no visible label.

<button>
  <svg aria-hidden="true">...</svg>
  <span class="sr-only">Close dialog</span>
</button>
