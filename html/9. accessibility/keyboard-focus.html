Keyboard Navigation & Focus Management

Tab Order

By default, keyboard focus moves through these elements in DOM order:

Natively focusableNotes
<a href>Links with an href
<button>Unless disabled
<input>, <select>, <textarea>Unless disabled
<details>The summary is focusable
Any element with tabindex="0"Added to natural tab order

tabindex Values

ValueEffect
tabindex="0"Add any element to the natural tab order (at its DOM position)
tabindex="-1"Programmatically focusable with .focus() but skipped by Tab key — used in JS-managed widgets
tabindex="N" (N > 0)Avoid — creates unpredictable tab order; use DOM order instead

Focus Ring — Never Remove Without Replacement

/* BAD — makes keyboard navigation invisible */
:focus { outline: none; }

/* GOOD — only hide for mouse users, keep for keyboard */
:focus:not(:focus-visible) { outline: none; }
:focus-visible { outline: 3px solid #2e7d32; outline-offset: 2px; }

Skip Navigation Link

The first focusable element on every page should be a link that jumps past repeated navigation to main content. It's invisible until focused (press Tab on this page to see it).

<a href="#main-content" class="skip-link">Skip to main content</a>
